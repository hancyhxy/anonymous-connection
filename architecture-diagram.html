Software architecture · three layers

Data flow
1
Identity Layer Avatar
Defines how each person looks on the device and on the wall. Each user builds their avatar on the phone when they first join — set once, seen everywhere afterwards.
Avatar definition · 4 fields
  • bodybody shape & pose
  • nicknameshort anonymous handle
  • bubblespeech-bubble quote (≤ 20 chars)
  • colorpalette index (5 tones)
~270 visual combinations from body × color (5 tones × 6 shades) × smile; bubble + nickname are free text on top — effectively unique per person.

2
Connection Layer Data collection & pairing
Collects each user's interest tags through a short questionnaire — right after the avatar in the setting stage. The server stores this data, then later uses it to pair people anonymously by similarity.
Interest tags · two-layer model
  • category e.g. movie, music, sport
  • subcategory e.g. movie → sci-fi, drama
Score = shared categories × 33% → maps to 0%, 33%, 67%, 100%. Subcategories sharpen the common-interest list but don't change the score.

→ Server (stores)
→ Wall layout by similarity
3
Trigger Layer Tap → similarity → icebreaker
When two devices tap, the server compares their interest arrays, picks the common ones, and returns a topic prompt — the icebreaker.
Match animation · 3 screens in sync
  • screen 1similarity % (e.g. 67%)
  • screen 2common interests
  • screen 3topic prompt (icebreaker)
Triggered by tap, paced by button — the reader presses to move from one screen to the next.
