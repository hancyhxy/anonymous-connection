# User Journey Map — Hello Stranger（pip / version2）

## 用户信息（User Info）

**姓名：** Mia
**背景：** 23 岁，UTS 在读硕士，刚来 Sydney 半年。喜欢参加 meetup 和校园活动，但每次进场之后都不太知道怎么开口，通常默默站在饮料桌旁刷手机。对"聊天 icebreaker"本身有点社交焦虑。
**一句话：** *"我其实想认识人，但不想自己硬着头皮上去搭话。"*

## 情境（Scenario）

Mia 进入一个线下社交活动现场。活动方提供了 Hello Stranger 系统：现场摆着一块 collective 大屏，每位参与者会拿到一台带屏幕的小设备（挂牌 / 手持）。Mia 扫码进入网页，设置自己的匿名 Avatar 并选择感兴趣的话题标签，然后通过大屏和手里的设备，尝试在现场找到一个可以搭话的人。

## 期望（Expectations）

- 不用主动破冰，也能找到跟自己话题接近的人
- 保持匿名感，不想一开场就暴露太多个人信息
- 希望有一个"被确认"的动作，让搭话的那一刻不尴尬
- 希望找到人之后，系统能给她一点话题引子，不用自己从零开始想

## Journey Map

| | **Stage 1：入场与设置** | **Stage 2：观察** | **Stage 3：找人** | **Stage 4：Tap 确认** | **Stage 5：解锁相似 Topic** |
|---|---|---|---|---|---|
| **目标 Goals** | 完成个人初始化：拿到一个匿名但属于自己的身份 | 通过 collective 视角理解"现场关系正在怎么变" | 把屏幕上的线索对应到现场的人，缩小搭话的候选范围 | 对目标对象发起一次明确的、双方都能感知到的确认 | 基于共同兴趣展开一段短暂但有内容的对话 |
| **行动 Actions** | - 扫码进入网页<br>- 配置 Avatar（形象 / 颜色 / 装饰）<br>- 勾选若干 interest topics<br>- 提交后，手里的设备同步显示个人内容 | - 抬头看大屏<br>- 观察 Avatar 的排序、位置和关系线变化<br>- 尝试猜测"为什么是这个排序"<br>- 把自己放到这个 collective 关系里定位 | - 对照大屏线索和手里设备提示<br>- 在人群中走动<br>- 观察别人设备上露出的 Avatar<br>- 锁定一个候选目标 | - 靠近目标对象<br>- 发起 tap 动作（设备靠近 / 点击）<br>- 等待两台设备同步反馈<br>- 完成一次确认 | - 查看设备上弹出的 shared topics<br>- 选一个话题作为开场白<br>- 进行一段 3–5 分钟的面对面对话<br>- 对话结束后自然退出，等待下一轮 |
| **想法 Thoughts** | - "还好是匿名的，不用填真名。"<br>- "勾兴趣这步挺像 Tinder，但感觉更轻。"<br>- "我的 Avatar 有点可爱，可以。" | - "那几个头像一直靠在一起，是因为兴趣重合吗？"<br>- "我的头像好像在往某一边移动？"<br>- "我好像不是孤立的一个点。" | - "屏幕上那个往我这边移的头像，是穿黄色外套的那个人吗？"<br>- "我要不要直接走过去？还是再等一下？"<br>- "万一我认错人就尴尬了。" | - "tap 下去如果对方没反应怎么办？"<br>- "啊，两台设备都亮了，那就是对上了。"<br>- "好，是这个人没错。" | - "我们居然都写了 film photography，这就可以聊了。"<br>- "比我自己硬破冰轻松太多了。"<br>- "聊完之后要不要再去看一次大屏？" |
| **痛点 Pain Points** | - 不确定选了 topic 之后系统会怎么用这些数据<br>- Avatar 选项如果太多会让设置变成门槛<br>- 初次进入不知道接下来会发生什么 | - 大屏变化背后的逻辑不透明，容易看不懂<br>- 如果人很多，自己的 Avatar 会淹没在群体里<br>- "我该看什么"没有明确提示 | - 大屏是 collective 视角，现场是物理空间，两者对不上位置关系<br>- 候选人不止一个时，很难判断"该选谁先 tap"<br>- 害怕走过去之后对方其实没看到自己 | - tap 的动作是否被识别，需要即时反馈<br>- 如果 tap 失败，社交成本很高<br>- 对方不愿意 tap 回来的情况没有兜底 | - 解锁的 topic 如果过于泛（比如"music"）就不是好引子<br>- 对话自然结束的时机很难把握<br>- 结束之后不清楚能不能和同一个人再 connect 一次 |
| **情绪 Emotions** | 好奇 / 轻度投入 | 好奇 / 想破解规则 | 兴奋 / 有一点试探的不安 | 紧张 / 终于确认的明确感 | 放松 / 被认可的安全感 |
| **触点 Touchpoints** | **线上：** 手机网页（扫码入口、Avatar 设置、topic 选择）<br>**硬件：** 手里的一体屏设备开始同步个人内容 | **线下：** collective 大屏 / 投影<br>**线上：** 后台匹配逻辑实时推算关系 | **硬件：** 其他参与者的手持设备（Avatar 公开展示层）<br>**线下：** 活动现场的物理空间 | **硬件：** 自己和对方的两台设备（tap 输入 + 反馈）<br>**线上：** 后台同步两端状态 | **硬件：** 两台设备的 shared topics 界面<br>**线下：** 面对面的真实对话 |
| **机会 Opportunities** | - 提供 3 套快速 Avatar 预设，降低设置时间<br>- 在设置页用一句话解释"topic 会怎么被用"，降低匿名焦虑<br>- 进入后给一个 30 秒的 onboarding 提示，告诉用户接下来看大屏 | - 为大屏加一个"为什么这样排"的轻量图例<br>- 高亮当前用户自己的 Avatar 位置，减少群体中迷失感<br>- 用渐变动画呈现变化过程，而不是瞬间跳变 | - 在手持设备上加一个"方向指示"或"距离感"提示，把 collective 坐标翻译成身体空间<br>- 限制同时高亮候选人的数量（比如最多 3 个），降低选择负担<br>- 给每个候选一个"新鲜度"标记，避免总是推同一个人 | - tap 成功时用明显的震动 + 光效双反馈<br>- tap 失败时给出轻量、非羞辱式的提示（如"对方还没准备好"）<br>- 支持对方 tap back，形成双向确认 | - 按兴趣重合度排序 shared topics，把最 specific 的话题放第一<br>- 给每个 shared topic 配一个 starter question，真的帮用户开口<br>- 在对话结束后允许两人选择"保留匿名收藏"，让这次连接可以被记住 |

## 情绪曲线（Emotion Curve）

```
自信          |                                                              *****
              |                                                         ****
安心          |                                                    ****
              |                              ***                ***
投入          |                         ****    **             **
              |                    ****          **          **
中性          |               ****                 **      **
              |          ****                        **  **
不确定        |     ****                               **
              |****
焦虑          |
              +-------|------------|-------------|------------|--------------->
               入场设置       观察 collective     找人         Tap 确认        解锁 topic
```

整条曲线是 **"好奇 → 微微不安 → 再度投入 → 稳定上扬"** 的走势：Mia 进场时是好奇和投入，观察阶段有一次认知负担的小谷（试图看懂大屏），找人和 tap 的过程带着试探感，确认成功的那一刻是情绪的主要转折点，最后解锁 topic + 面对面聊天让整条曲线稳定上抬。
